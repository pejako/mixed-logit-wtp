# mixedlogitr — R parity layer

This is the R companion to the Python `mixedlogit` package. It does not
re-implement the SML estimator — that is the wrong place to spend
maintenance effort. Instead, it:

1. Reads the canonical synthetic dataset that the Python side generates
2. Fits MNL and MXL via the standard `mlogit` package
3. Computes the same elasticity and WTP outputs
4. Compares estimates side-by-side with the Python results

The point is **cross-language reproducibility**. Two independent
implementations agreeing on the same data is a much stronger correctness
signal than either one in isolation.

## Quickstart

From the project root:

```bash
# Generate the shared dataset and Python results JSON
cd python
python -m mixedlogit.export_csv          # -> r/data/synthetic_choices.csv
python scripts/export_results.py          # -> r/data/python_*_results.json
```

In R:

```r
# Install dependencies (one-time)
install.packages(c("mlogit", "dfidx", "jsonlite", "testthat"))

# From the r/ directory
setwd("path/to/mixed-logit-choice-lab/r")
source("R/data.R")
source("R/estimate.R")
source("R/elasticity.R")
source("R/wtp.R")
source("R/parity.R")

df <- load_synthetic_data()
mxl <- fit_mxl_r(df, n_draws = 200)
print(mxl$coefficients)

# Cross-language parity
cmp <- compare_with_python(mxl, "data/python_mxl_results.json")
print(cmp)
```

## Status

**v0.2-pre.** The R code is written against the documented `mlogit` v1.1+
API and is structurally aligned with the Python implementation
field-for-field. The CSV interchange has been smoke-tested in base R; the
parse and load paths work end-to-end. The full parity tests run in CI
where `mlogit` and dependencies install cleanly.

Run locally with:

```r
testthat::test_dir("tests/testthat")
```

Tests that require `mlogit` will skip cleanly if it isn't installed
(`skip_if_not_installed("mlogit")`), so the test runner works regardless
of environment.

## Tolerances

The parity tests assert:

- **MNL coefficients** within 5% of Python (MNL is deterministic; the
  gap should typically be much smaller — Python uses BFGS with analytic
  gradient, R's `mlogit` uses Newton-Raphson, both converge to the same
  optimum)
- **MXL means** within 5% of Python
- **MXL standard deviations** within 10% of Python (SML simulation noise
  affects the SDs more than the means, because the SDs are estimated
  from the *spread* of the per-draw predictions rather than the central
  tendency)

These are operating tolerances, not theoretical bounds. If you see gaps
larger than this, suspect: (a) a different DGP seed, (b) a different
`n_draws`, (c) a bug.
